# AGENTS.md

You are building an academic thesis repository for a University of Patras ECE diploma/undergraduate thesis on optimal solution algorithms for the Rubik's Cube.

## Always read first

Before making changes, read these files:

1. `docs/goal.md`
2. `docs/acceptance.md`
3. `docs/requirements_from_brief.md`
4. `specs/topic_brief.pdf`, if accessible
5. `docs/progress.md`, if it already exists

## Source priority

Use this order when resolving conflicts:

1. The uploaded thesis topic brief in `specs/topic_brief.pdf`
2. Current official University of Patras / ECE thesis rules, if verified from official sources
3. `docs/goal.md`
4. `docs/acceptance.md`

## Academic integrity rules

- Do not invent citations, papers, authors, publication years, DOIs, URLs, benchmark values, theorem statements, or university requirements.
- Do not state that an algorithm is optimal unless the implementation and tests actually prove optimality for the reported cases.
- Do not state that arbitrary 3x3 Rubik's Cube positions are solved optimally unless this is implemented, benchmarked, and verified.
- Do not present approximate, timed-out, or lower-bound output as exact distance.
- Do not copy long passages from sources. Summarize in original words and cite properly.
- If information is not verified, mark it as a limitation or as a source that must be fetched and verified.

## Packaging and imports

- The canonical package is `src/rubik_optimal` (a `src/` layout). It is the
  single source of truth for the public API.
- Install editable from this repository root: `pip install -e ".[dev]"`, or run
  with `PYTHONPATH=src`. Both make `import rubik_optimal` resolve to `src`.
- `rubik_optimal/__init__.py` at the repo root is only a thin raw-checkout shim.
  It appends `src/rubik_optimal` to its `__path__` and re-exports the canonical
  public API dynamically. Do **not** turn it back into a hand-maintained symbol
  list, and do not let it shadow or diverge from `src`.
- The default install has no runtime dependencies. The pip `kociemba` package
  (GPLv2) is an opt-in extra (`pip install -e ".[baselines]"`) used only by the
  non-exact `solve_kociemba_adapter` baseline. Keep GPL code out of the default
  install. See `THIRD_PARTY_NOTICES.md` and `REPRODUCIBILITY.md`.

## Engineering rules

- Prefer reproducible scripts over manually generated output.
- Keep code and thesis claims synchronized.
- Every benchmark value in the thesis must come from saved result files.
- Every result file must be generated by a script.
- Every figure/table in the thesis must be traceable to a script or documented source file.
- Add tests before or alongside non-trivial algorithm implementation.
- Do not remove passing tests to hide a bug.
- Do not remove working verified functionality when attempting a more ambitious extension.

## Thesis language

The thesis must be written mainly in formal Greek academic style, with:

- Greek abstract;
- English abstract;
- Greek chapter headings;
- precise technical terminology;
- consistent notation;
- clear distinction between exact, approximate, heuristic, and bounded results.

## Definition of completion

The project is complete only when:

1. The code runs.
2. Tests pass.
3. Benchmark scripts run and produce saved results.
4. The LaTeX thesis builds.
5. Citations resolve.
6. Claims in the thesis match implemented evidence.
7. Limitations are documented honestly.
8. `docs/acceptance.md` is satisfied.

If any of these are not true, do not claim completion.
